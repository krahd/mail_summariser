Getting Started

  1. Pick a starting view: Use Quick Filters in the left rail, then refine with Advanced Query.
  2. Generate digest: Click Generate Digest to create a new summary job from the active filters.
  3. Review context: Inspect the Summary Canvas and message stack; click any message row for full body preview.
  4. Apply scoped actions: Use the right action panel to choose mark/tag/email operations, then apply to the current job.

Tabs

  • Main: Studio view with quick filters, digest canvas, message review, and scoped actions.
  • Log: Action timeline with search, status filtering, and undoable-only filter.
  • Settings: Configure backend connection, LLM provider, email settings, and Ollama models.
  • Help: This page.

Actions

  • Mark messages read: Marks every message in the active summary job as read.
  • Tag as summarised: Applies the configured summary tag to messages in the active job.
  • Email summary: Sends the summary text to the configured recipient address.
  • Undo Last: Reverses the latest reversible action from the current workspace state.

Settings

Configure mail basics on the main settings screen, then open Advanced Settings for provider prompts, model tools, backend controls, and operator actions.

  • Sample Mailbox: Toggle between resettable sample mail and a real IMAP/SMTP account.
  • Test Connection: Validate the current settings against the sample mailbox or the configured IMAP/SMTP servers before saving.
  • LLM Provider: Choose between Ollama (local), OpenAI (remote), or Anthropic (remote).
  • Ollama lifecycle: Choose whether Ollama starts with mail_summariser, starts only when needed, or is stopped automatically on exit.
  • Stop mail_summariser: Shut down the connected backend from the Settings tab.
  • Backend URL: Point to a different backend service if needed.
  • Backend API Key: Optional authentication for the backend (if enabled).
