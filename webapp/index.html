Getting Started

  1. Search criteria: Fill in the search fields on the main tab. You can filter by keyword, sender, tags, and more.
  2. Run a summary: Click Create Summary to search for matching messages and generate a summary.
  3. Review results: Check the summary text and review the list of messages that were included.
  4. Take action: Use the buttons below the summary to mark messages read, add tags, or email the summary.

Tabs

  • Main: The main interface. Search for messages and create summaries.
  • Log: View the history of actions performed. You can undo recent actions here.
  • Settings: Configure backend connection, LLM provider, email settings, and Ollama models.
  • Help: This page.

Actions

  • Mark Read: Mark all messages in the summary result as read.
  • Tag Summarised: Add the configured tag to all messages in the result.
  • Email Summary: Send the summary text to the configured recipient email.
  • Undo Last: Reverse the last action (available in the Log tab).

Settings

Configure IMAP/SMTP connection details, choose an LLM provider, and set up Ollama if using local inference.

  • Dummy Mode: Toggle between the built-in test mailbox and a real IMAP/SMTP account.
  • Test Connection: Validate the current settings against dummy mode or the configured IMAP/SMTP servers before saving.
  • LLM Provider: Choose between Ollama (local), OpenAI (remote), or Anthropic (remote).
  • Ollama lifecycle: Choose whether Ollama starts with Mail Summariser, starts only when needed, or is stopped automatically on exit.
  • Stop Mail Summariser: Shut down the connected backend from the Settings tab.
  • Backend URL: Point to a different backend service if needed.
  • Backend API Key: Optional authentication for the backend (if enabled).
